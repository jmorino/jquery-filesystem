jQuery.sync(sDBName, sRemoteIDPath, oConfig, eConflictPolicy, iVersion)
Constructor for a new jQuery.sync.
Parameters:
{string} sDBName
name of the database
{string} sRemoteIDPath
a path to server's ID in object properties (mandatory to create index)
{object} oConfig
configuration object that contains actual configuration
{function} oConfig.create
function to send locally-created records to server. It takes a record object as only parameter.
{function} oConfig.requestSinceDate
function to fetch records from server since a Date. It takes a Date object as only optional parameter. If parameter is not provided, it fetches all records.
{function} oConfig.update
function to send locally-updated records to server. It takes a record object as only parameter.
{function} oConfig.remove
function to send locally-deleted records to server ("delete" is not used as it is a reserved keyword in JavaScript).. It takes a record object as only parameter.
{object} oConfig.indexes
map of strings representing indexes to create. Each entry is of type index_name: path_to_value
{$.sync.CONFLICT_POLICY} eConflictPolicy
determines how conflicts should be resolved: always trust the server (SERVER_IS_ALWAYS_RIGHT) or the client (CLIENT_IS_ALWAYS_RIGHT - default).
{integer} iVersion
version number of the database
